01 Overview

데이터 수집, 저장, AI 분석, 경고, 원격 알림까지 통합

기존 개별 실습을 하나의 시스템으로 연결합니다. 단순히 센서값이 기준을 넘었는지 확인하는 것이 아니라, 정상 환경 패턴을 학습한 뒤 현재 데이터가 평소와 다르면 AI가 이상 상태로 판단하도록 구성합니다.

Project 라즈베리파이를 활용한 AIoT 기반 실내 환경 데이터로거 및 이상 알림 시스템 구현

Development of an AIoT-based Indoor Environmental Data Logger and Anomaly Alert System Using Raspberry Pi

Repository aiot-env-anomaly-logger

새 주제에 맞춘 GitHub 레포지토리명 추천입니다.

Core Flow 수집 → 저장 → AI 분석 → 알림

센서 데이터가 CSV 로그와 AI 판단, 웹 대시보드, Telegram 알림으로 이어집니다.

Difference 조건문 + AI 역할 분리

위험 가스값은 조건문으로 즉시 DANGER, 평소와 다른 변화는 AI가 ANOMALY로 감지합니다.

02 Final Features

최종 기능 구성

센서 측정부터 웹 모니터링과 Telegram 원격 알림까지 과제 조건을 자연스럽게 만족하는 구조입니다.

기능설명
온습도 측정DHT11 또는 DHT22 센서로 실내 온도와 습도를 측정합니다.
가스/연기 감지MQ-2 센서로 가스 또는 연기 상태를 감지합니다.
아날로그값 확장PCF8591 ADC로 MQ-2 AOUT 값을 읽어 가스값 변화량을 분석합니다.
데이터 저장timestamp, temperature, humidity, gas_value, 변화량, AI 결과를 CSV에 저장합니다.
AI 이상탐지Isolation Forest로 정상 환경 패턴과 다른 센서값 조합을 ANOMALY로 판단합니다.
로컬 경고LED, 부저, LCD로 NORMAL, WARNING, DANGER 상태를 표시합니다.
웹 대시보드Flask에서 현재 센서값, AI 결과, Anomaly Score, 최근 로그를 확인합니다.
Telegram 알림WARNING 또는 DANGER 발생 시 원격 알림 메시지를 전송합니다.

03 AI Anomaly Detection

Isolation Forest로 평소와 다른 실내 환경 변화를 감지

AI는 온도, 습도, 가스값의 현재 값과 변화량을 기반으로 현재 환경이 평소 정상 상태와 비슷한지, 아니면 비정상적인 패턴인지 판단합니다. 

features = [
  "temperature",
  "humidity",
  "gas_value",
  "temp_change",
  "humidity_change",
  "gas_change",
]

model = IsolationForest(contamination=0.05)
model.fit(normal_data[features])
joblib.dump(model, "models/isolation_forest.pkl")
AI 입력값설명
temperature현재 온도
humidity현재 습도
gas_value현재 MQ-2 센서값
temp_change직전 측정값 대비 온도 변화량
humidity_change직전 측정값 대비 습도 변화량
gas_change직전 측정값 대비 가스값 변화량
AI 판단 예시 27℃ · 63% · Gas 280 → ANOMALY

습도 80%나 가스값 500 같은 위험 기준에는 도달하지 않았더라도, 평소 정상 범위에서 크게 벗어나면 AI가 이상 상태로 판단할 수 있습니다.

Safety Rule 가스 위험 기준 초과 → 즉시 DANGER

가스/연기처럼 위험도가 높은 상황은 AI 판단만 기다리지 않고, 조건문으로 즉시 DANGER 상태를 처리합니다.

04 Hardware

DHT 센서, MQ-2, PCF8591, LCD, LED, 부저 구성

처음에는 MQ-2 DOUT으로 가스 감지 여부를 확인하고, 이후 PCF8591 ADC를 이용해 AOUT 아날로그값을 읽는 방식으로 확장합니다.

사용 부품

  • Raspberry Pi 4B 또는 5
  • DHT11 또는 DHT22 온습도 센서
  • MQ-2 가스 센서
  • PCF8591 ADC
  • LED 3개, 330Ω 저항
  • 능동부저
  • 1602 LCD
  • 브레드보드, 점퍼 케이블

GPIO 연결 예시

  • DHT DATAGPIO 4
  • MQ-2 DOUTGPIO 17
  • MQ-2 AOUTPCF8591 AIN0
  • PCF8591 SDARaspberry Pi SDA
  • PCF8591 SCLRaspberry Pi SCL
  • 초록 LEDGPIO 20
  • 노랑 LEDGPIO 21
  • 빨강 LEDGPIO 26
  • 부저GPIO 18
  • LCD 1602I2C 또는 GPIO 방식

08 Demo Plan

2~3분 영상에서 정상, 이상, 위험, 로그 저장을 보여주기

불을 사용하지 않고 입김, 젖은 휴지, 손소독제 또는 알코올 솜을 활용해 안전하게 변화 상황을 연출합니다.

장면 1프로젝트 소개

전체 회로, 라즈베리파이, DHT 센서, MQ-2, LED, 부저, LCD, 웹 대시보드를 보여줍니다.

장면 2정상 상태

초록 LED ON, 부저 OFF, LCD NORMAL, 웹 대시보드 NORMAL, CSV 로그 저장을 확인합니다.

장면 3습도 변화 시연

입김 또는 젖은 휴지를 센서 근처에 두어 습도 상승, ANOMALY, WARNING, Telegram 알림을 보여줍니다.

장면 4가스 변화 시연

손소독제나 알코올 솜을 MQ-2 근처에 접근시켜 가스값 상승, DANGER, 부저, 긴급 알림을 보여줍니다.

장면 5로그 저장 확인

sensor_log.csv에 시간, 온도, 습도, 가스값, AI 판단 결과, 상태가 저장된 것을 보여줍니다.

Report보고서 구성

실험 목적, 이론, 환경 및 절차, 결과, GitHub, YouTube, 참고문헌 흐름으로 작성합니다.

09 Conclusion

단순 센서 실습보다 완성도 높은 AIoT 실내 환경 모니터링 시스템

이 프로젝트는 온습도 센서, MQ-2 가스 센서, 브레드보드, LED, 부저, LCD를 활용하고, Flask 웹 대시보드와 Telegram 알림, AI 이상탐지 모델까지 포함하기 때문에 AIoT 설계실습 기말 프로젝트로 적합합니다.

핵심 차별점은 단순 조건문 기반 경고 시스템이 아니라, 정상 환경 데이터 패턴을 학습한 AI 모델이 평소와 다른 센서값 변화까지 감지하는 실내 환경 모니터링 시스템이라는 점입니다.
