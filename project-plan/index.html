01 Overview

현재 습도 조건문이 아니라 10분 후 습도 예측으로 제어

단순히 현재 습도가 낮으면 가습기를 켜는 방식이 아니라, 실내 온습도, 외부 날씨 API, 최근 습도 변화량, 물 상태를 AI 모델 입력값으로 사용합니다. 시스템은 AI가 예측한 미래 습도를 기준으로 가습기 ON/OFF를 결정합니다.

Project 라즈베리파이를 활용한 AI 기반 실내 습도 예측 스마트 자동 가습 시스템 구현

Development of an AI-based Indoor Humidity Prediction Smart Humidifier System Using Raspberry Pi

Repository ai-smart-humidifier

이 프로젝트 전용 GitHub 레포지토리명입니다.

Core Logic AI 예측값 기준 제어

현재 습도 대신 predicted_humidity_10min을 기준으로 릴레이를 제어합니다.

Safety LOW WATER 우선 정지

02 Final Features

최종 기능 구성

센서 입력, API 연동, AI 예측, 하드웨어 제어, 상태 표시, 데이터 기록까지 하나의 흐름으로 구성합니다.

기능설명
실내 온습도 측정DHT22 센서로 현재 실내 온도와 습도를 수집합니다.
외부 날씨 확인OpenWeatherMap API로 외부 온도와 외부 습도를 가져옵니다.
AI 습도 예측RandomForestRegressor로 10분 후 실내 습도를 예측합니다.
자동 가습 제어예측 습도가 낮으면 릴레이를 통해 가습기 전원을 켭니다.
물 부족 감지XKC-Y25-V 비접촉식 수위센서를 물병 외부에 부착합니다.
안전 정지LOW WATER 상태에서는 AI 예측과 관계없이 가습기를 강제로 끕니다.
LCD 표시현재 습도, 예측 습도, 가습기 상태를 LCD1602에 표시합니다.
LED/부저 알림정상, 가습, 물부족 상태를 LED와 부저로 표시합니다.
웹 대시보드Flask로 전체 상태를 확인하는 대시보드를 구성합니다.
데이터 기록CSV로 센서값, 예측값, 시스템 상태를 저장합니다.

03 AI Role

AI는 가습기 ON/OFF가 아니라 10분 후 실내 습도를 예측

이 프로젝트에서 AI의 핵심 역할은 미래 습도 예측입니다. 제어 로직은 AI가 출력한 predicted_humidity_10min 값을 기준으로 가습기를 작동시킵니다. 

features = [
  "indoor_temp",
  "indoor_humidity",
  "outdoor_temp",
  "outdoor_humidity",
  "humidity_change_5min",
  "humidifier_state",
  "water_state",
]

target = "predicted_humidity_10min"
model = RandomForestRegressor()
model.fit(X_train, y_train)
joblib.dump(model, "model/humidity_predictor.pkl")
입력값설명
indoor_temp현재 실내 온도
indoor_humidity현재 실내 습도
outdoor_temp외부 온도
outdoor_humidity외부 습도
humidity_change_5min최근 5분간 실내 습도 변화량
humidifier_state현재 가습기 상태, ON=1/OFF=0
water_state물 상태, 물 있음=1/물 부족=0
시연 핵심 장면 현재 52% → AI 예측 43% → 가습기 ON

현재 습도는 적정 범위에 가깝지만, 외부 습도와 최근 습도 감소 추세를 반영한 AI 모델이 10분 후 습도 하락을 예측하여 가습기를 자동으로 작동시키는 장면을 보여줍니다.

AI 검증 MAE + Feature Importance

MAE로 예측 습도와 실제 습도의 평균 오차를 확인하고, Feature Importance로 현재 습도, 최근 5분 변화량, 외부 습도 등이 예측에 얼마나 영향을 주는지 분석합니다.

04 Hardware Structure

물병형 초음파 가습기, 수위센서, 릴레이 전원 제어 구조

가습기 모듈은 물병 위쪽 캡 부분에 고정합니다. 필터심 끝은 항상 물에 닿아 있어야 하며, XKC-Y25-V 수위센서는 물병 안이 아니라 바깥쪽에 부착합니다.

Humidifier Structure 물병 안의 물 → 필터심 → 초음파 진동판 → 미스트 배출

가습기 모듈은 물병 위쪽에 위치하고, 필터심이 물을 흡수해 상단 초음파 진동판으로 전달합니다. 미스트는 위쪽으로 배출되며 필터심 끝은 항상 물에 닿아 있어야 합니다.

Water Level XKC-Y25-V 비접촉식 수위센서

센서 위치까지 물이 있으면 WATER OK, 센서 위치보다 물이 낮아지면 LOW WATER로 판단합니다. LOW WATER 상태에서는 가습기 OFF, 빨강 LED ON, 부저 ON, LCD LOW WATER 표시를 수행합니다.

05 Power Control

가습기 기본 케이블은 자르지 않고 USB 연장 케이블만 릴레이 제어

초음파 가습기 모듈은 라즈베리파이 5V 핀에 직접 연결하지 않습니다. 외부 USB 전원과 릴레이 접점을 사용해 가습기 전원을 안전하게 제어합니다.

01USB 충전기

가습기 모듈용 외부 5V 전원을 공급합니다.

02USB 연장 케이블

가습기 기본 케이블 대신 연장 케이블 하나만 잘라 사용합니다.

03릴레이 COM/NO

빨강 +5V 선만 릴레이 접점에 연결해 전원을 끊고 켭니다.

04GND 유지

검정 GND는 그대로 연결하고 흰색/초록 데이터선은 각각 절연합니다.

05Micro 5-pin

마이크로 5핀 케이블을 통해 가습기 모듈에 전원을 전달합니다.

06Relay ON/OFF

릴레이 OFF면 가습기 OFF, 릴레이 ON이면 가습기 ON 상태가 됩니다.

06 Control & Output

제어 우선순위와 LCD/대시보드 출력

물 부족 여부를 가장 먼저 확인하고, 그 다음 센서/API 오류, 마지막으로 AI 예측 습도를 기준으로 제어합니다.

08 Demo Plan

3~5분 시연 영상에서 AI 예측 기반 제어를 확실히 보여주기

실제 습도 변화는 느리기 때문에 영상에서는 데모 모드를 사용합니다. 단, 실제 시스템은 10분 후 실내 습도를 예측하도록 설계되었다는 점을 설명합니다.

장면 1전체 하드웨어 소개

라즈베리파이, 브레드보드, DHT22, LCD1602, 릴레이, LED/부저, 수위센서, 물병형 가습기 모듈을 보여줍니다.

장면 2물병형 가습기 구조 설명

가습기 모듈은 물병 위쪽에 있고, 필터심이 물을 빨아올리며, 수위센서는 바깥쪽에 붙는다는 점을 설명합니다.

장면 3정상 상태

AI 예측 습도 45~60%, 가습기 OFF, 초록 LED ON, LCD NORMAL 상태를 보여줍니다.

장면 4AI 예측 기반 가습 ON

현재 습도 52%, 예측 습도 43% 상황을 만들어 10분 후 건조 예상으로 가습기가 켜지는 장면을 보여줍니다.

장면 5물 부족 감지

물이 센서 아래로 내려가면 LOW WATER, 가습기 OFF, 빨강 LED, 부저, LCD 경고가 나오는 장면을 보여줍니다.

장면 6Flask 대시보드

실내 습도, 외부 습도, AI 예측 습도, 물 상태, 가습기 상태, 시스템 상태를 웹에서 확인합니다.
