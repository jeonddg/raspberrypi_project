01 Overview

프로젝트 한눈에 보기

이 프로젝트는 센서 데이터 수집, AI 예측, GPIO 제어, 웹 대시보드까지 연결되는 작은 규모의 완성형 AIoT 시스템을 목표로 합니다.

Goal 예측 기반 가습 제어

현재 습도뿐 아니라 가까운 미래의 습도 변화를 예측해 더 안정적으로 가습기를 제어합니다.

Data 실내 + 외부 환경 데이터

DHT 센서, 물 수위 센서, OpenWeatherMap API 데이터를 함께 사용합니다.

Output 가습기, LCD, LED, 부저

릴레이 모듈로 가습기를 제어하고 LCD와 알림 장치로 상태를 표시합니다.

Portfolio GitHub Pages 문서화

기획, 구조, 구현 흐름을 웹 페이지로 정리해 포트폴리오 링크로 활용합니다.

02 Problem & Approach

왜 예측 기반 제어가 필요한가

일반적인 자동 가습기는 현재 습도가 기준값보다 낮으면 켜지고, 기준값 이상이면 꺼지는 단순한 방식으로 동작합니다. 하지만 실제 실내 습도는 외부 날씨, 온도, 최근 변화 속도, 가습기 동작 이력에 따라 계속 달라집니다.

일반 제어 방식

  • 현재 습도 기준으로만 ON/OFF 결정
  • 습도 변화 추세를 반영하기 어려움
  • 건조해진 뒤에야 가습 시작

AI 예측 제어 방식

  • 10분 후 습도를 예측해 사전 대응
  • 외부 날씨와 최근 변화량까지 반영
  • 물 부족과 과습 상황에서는 안전 정지

04 Core Features

주요 기능

하드웨어 제어와 데이터 기반 판단을 함께 보여줄 수 있도록 기능을 모듈 단위로 나눕니다.

01

실내 온습도 측정

DHT11 또는 DHT22 센서로 현재 실내 온도와 습도를 측정하고, AI 모델 입력값으로 사용합니다.

02

외부 날씨 API 연동

OpenWeatherMap API를 활용해 외부 온도, 외부 습도, 날씨 상태를 수집합니다.

03

물 수위 감지

Water Detection Sensor와 PCF8591 ADC로 물 부족 상태를 감지하고 가습기를 강제 정지합니다.

04

AI 습도 예측

실내외 환경 데이터와 최근 변화량을 바탕으로 10분 후 습도를 예측합니다.

05

릴레이 기반 자동 제어

예측 습도와 물 수위 상태를 기준으로 초음파 가습 모듈 전원을 제어합니다.

06

상태 표시 및 대시보드

1602 LCD, LED, 부저, Flask 웹 대시보드로 시스템 상태를 직관적으로 확인합니다.

05 AI Model Design

AI 모델 설계

초기 모델은 구현 난이도와 설명 가능성을 고려해 Linear Regression, Decision Tree, Random Forest를 비교하고, 최종적으로 Random Forest Regressor를 우선 적용합니다.

입력 변수 설명 활용 목적
indoor_temp 현재 실내 온도 습도 변화에 영향을 주는 실내 환경 기준
indoor_humidity 현재 실내 습도 예측의 기본 출발값
outdoor_temp 외부 온도 실내 건조 가능성 보정
outdoor_humidity 외부 습도 날씨에 따른 실내 습도 변화 반영
humidity_change_5min 최근 5분 습도 변화량 습도가 내려가는 속도 또는 회복 속도 파악
humidifier_state 가습기 현재 동작 상태 가습 중인지 여부를 예측에 반영
water_level 물 수위 상태 안전 제어 조건으로 사용
예측 습도 45% 미만

물 수위가 충분하면 가습기를 ON으로 전환합니다.

예측 습도 45~60%

정상 범위로 판단하고 현재 상태를 유지하거나 OFF로 둡니다.

예측 습도 60% 초과

과습 방지를 위해 가습기를 OFF로 전환합니다.

물 부족 감지

가습기를 강제 OFF하고 LED와 부저로 경고합니다.
