01 Project Overview

제품처럼 보이지만, 핵심은 프로젝트 기획서

이 페이지는 완성된 제품 홍보 페이지가 아니라, AI 스마트 가습기 프로젝트의 목적, 시스템 구조, 데이터 흐름, 구현 계획을 포트폴리오용으로 정리한 기획서입니다.

Goal 예측 기반 자동 가습

10분 후 실내 습도를 예측해 가습기 동작을 미리 제어합니다.

Device Raspberry Pi 제어

GPIO, 릴레이, 센서, LCD를 연결해 실제 하드웨어 제어까지 구현합니다.

Data 실내외 환경 데이터

DHT 센서, 물 수위 센서, OpenWeatherMap API 데이터를 함께 사용합니다.

Interface LCD + Flask Dashboard

현재 상태와 AI 예측 결과를 로컬 LCD와 웹 대시보드에서 확인합니다.

02 Problem & Goal

단순 기준값 제어의 한계를 줄이는 것이 목표

일반 자동 가습기는 현재 습도가 기준값보다 낮으면 켜지고, 기준값 이상이면 꺼지는 방식입니다. 하지만 실제 습도는 외부 날씨, 실내 온도, 가습기 동작 이력, 최근 변화 속도에 따라 계속 달라집니다.

기존 방식

  • 현재 습도 하나만 기준으로 판단
  • 습도 하락 추세를 미리 반영하기 어려움
  • 물 부족이나 센서 상황을 종합적으로 보기 어려움

제안 방식

  • 10분 후 습도 예측값을 기준으로 제어
  • 실내 데이터와 외부 날씨 데이터를 함께 반영
  • 물 부족과 과습 상황에서는 가습기를 안전하게 정지

03 How It Works

센서에서 대시보드까지 이어지는 데이터 흐름

기획 단계에서는 전체 흐름을 명확히 잡는 것이 중요하므로, 수집부터 제어까지의 단계를 먼저 정의합니다.

01 Sensor Input

DHT 센서와 물 수위 센서로 실내 상태를 측정합니다.

02 Weather Data

OpenWeatherMap API로 외부 온도와 습도를 가져옵니다.

03 CSV Logging

센서값과 가습기 상태를 시간 순서대로 저장합니다.

04 AI Prediction

모델이 10분 후 실내 습도를 예측합니다.

05 Relay Control

예측값과 안전 조건에 따라 가습기를 제어합니다.

06 Dashboard

LCD와 Flask 화면에 현재 상태를 표시합니다.

04 Product Features

제품 소개처럼 정리한 핵심 기능

아직 기획 단계이지만, 최종 구현 시 사용자가 보게 될 기능 단위로 정리했습니다.

Feature 01

AI 습도 예측

현재 실내 습도와 외부 환경 데이터를 바탕으로 10분 후 습도를 예측합니다.

Feature 02

자동 가습 제어

예측 습도가 낮아질 가능성이 있으면 릴레이로 가습 모듈을 동작시킵니다.

Feature 03

물 부족 감지

물 수위가 낮으면 가습기를 강제 정지하고 LED와 부저로 경고합니다.

Feature 04

과습 방지

예측 습도가 높을 경우 가습을 중단해 불필요한 동작을 줄입니다.

Feature 05

LCD 상태 표시

현재 습도, 예측 습도, 가습 상태, 물 부족 상태를 1602 LCD에 표시합니다.

Feature 06

Flask 웹 대시보드

센서값, API 데이터, AI 예측값, 시스템 상태를 웹 화면에서 확인합니다.

05 AI Model Plan

AI 모델 설계 계획

초기 구현에서는 설명 가능성과 구현 난이도를 고려해 Linear Regression, Decision Tree, Random Forest를 비교하고, 우선 적용 모델은 Random Forest Regressor로 계획합니다.

입력 변수 설명 사용 목적
indoor_temp 현재 실내 온도 실내 환경 변화 반영
indoor_humidity 현재 실내 습도 예측 기준값
outdoor_temp 외부 온도 건조 가능성 보정
outdoor_humidity 외부 습도 외부 날씨 영향 반영
humidity_change_5min 최근 5분 습도 변화량 습도 변화 속도 파악
humidifier_state 가습기 동작 상태 가습 중인지 여부 반영
water_level 물 수위 상태 안전 제어 조건

07 Development Roadmap

구현 단계

기획 단계에서 끝나지 않고 실제 구현으로 이어지도록 단계별 작업을 정리했습니다.

  1. 1기획 완료

    프로젝트 목적, 기능, 시스템 구조, 부품 구성을 정리합니다.

  2. 2하드웨어 테스트

    DHT 센서, 물 수위 센서, LCD, LED, 부저, 릴레이를 개별 테스트합니다.

  3. 3API 연동

    OpenWeatherMap API로 외부 온습도 데이터를 가져옵니다.

  4. 4데이터 수집

    센서값과 가습기 상태를 CSV로 저장해 학습 데이터를 구성합니다.

  5. 5AI 모델 구현

    습도 예측 모델을 학습하고 실시간 제어 프로그램에서 사용합니다.

  6. 6통합 제어

    예측값과 안전 조건에 따라 가습기 ON/OFF를 자동 결정합니다.

  7. 7대시보드 제작

    Flask 웹 화면에서 현재 상태와 예측 결과를 확인합니다.

  8. 8결과 정리

    회로 사진, 실행 화면, 시연 영상, README를 포트폴리오 형태로 정리합니다.

08 Success Criteria & Risk

완성 기준과 예상 리스크

기획서로서 프로젝트의 완료 기준과 위험 요소도 함께 명확히 정리합니다.

성공 기준

  • 실내 온습도 값을 안정적으로 수집한다.
  • 외부 날씨 API 데이터를 정상적으로 가져온다.
  • 10분 후 습도 예측값을 생성한다.
  • 예측 습도와 물 수위에 따라 가습기를 제어한다.
  • LCD와 Flask 대시보드에 상태를 표시한다.

예상 리스크와 대응

  • 센서값 오차는 이동 평균과 임계값 보정으로 완화한다.
  • 학습 데이터 부족은 초기 샘플 데이터로 먼저 검증한다.
  • 릴레이와 가습 모듈 전원은 안정성을 고려해 분리한다.
  • 물 수위 센서 노이즈는 반복 측정 후 평균값을 사용한다.
  • API 오류 시 최근 값을 임시로 사용하는 예외 처리를 둔다.
