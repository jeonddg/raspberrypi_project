# AI 기반 실내 습도 예측 스마트 자동 가습 시스템 기획서

## 1. 프로젝트 개요

### 프로젝트명

**라즈베리파이를 활용한 AI 기반 실내 습도 예측 스마트 자동 가습 시스템**

### 영문명

**AI-based Indoor Humidity Prediction Smart Humidifier System Using Raspberry Pi**

### 프로젝트 설명

본 프로젝트는 Raspberry Pi를 기반으로 실내 온습도, 물 수위, 외부 날씨 데이터를 수집하고, AI 모델을 이용하여 일정 시간 후의 실내 습도를 예측한 뒤 가습기를 자동으로 제어하는 AIoT 시스템이다.

일반적인 자동 가습기는 현재 습도 값만을 기준으로 동작하지만, 본 시스템은 현재 실내 습도뿐만 아니라 외부 습도, 온도, 최근 습도 변화량, 가습기 동작 상태, 물 수위 상태를 함께 고려한다. 이를 통해 단순 임계값 기반 제어보다 더 지능적인 가습 제어를 목표로 한다.

---

## 2. 프로젝트 목적

본 프로젝트의 목적은 실내 환경 데이터를 기반으로 가습기를 자동 제어하는 스마트 가습 시스템을 구현하는 것이다. 또한 OpenWeatherMap API를 활용하여 외부 날씨 데이터를 수집하고, AI 모델을 통해 10분 후 실내 습도를 예측함으로써 단순 조건문 방식이 아닌 예측 기반 제어 시스템을 구현하고자 한다.

이를 통해 다음과 같은 기술을 학습하고 적용한다.

- Raspberry Pi GPIO 제어
- 온습도 센서를 이용한 실내 환경 측정
- 수위 센서를 이용한 물 부족 상태 감지
- OpenWeatherMap API를 활용한 외부 날씨 데이터 수집
- AI 모델을 이용한 실내 습도 예측
- 릴레이 모듈을 이용한 초음파 가습 모듈 제어
- 1602 LCD를 활용한 상태 표시
- Flask 웹 대시보드를 통한 IoT 모니터링

## 3. 주제 선정 배경

실내 습도는 쾌적한 생활 환경과 건강 관리에 중요한 요소이다. 습도가 너무 낮으면 피부 건조, 호흡기 불편, 정전기 발생 등의 문제가 생길 수 있으며, 반대로 습도가 너무 높으면 곰팡이와 세균 번식의 원인이 될 수 있다.

기존의 자동 가습기는 보통 현재 습도 값이 기준 이하로 내려가면 가습기를 켜고, 기준 이상이 되면 끄는 단순 제어 방식을 사용한다. 그러나 실내 습도는 외부 날씨, 실내 온도, 최근 습도 변화량, 가습기 작동 여부 등에 따라 계속 변화한다.

따라서 본 프로젝트에서는 현재 습도만 기준으로 판단하지 않고, 여러 데이터를 종합하여 AI 모델이 미래 습도를 예측하도록 설계하였다. 이를 통해 가습기가 필요한 상황을 미리 판단하고, 불필요한 작동이나 과습을 줄이는 스마트 자동 가습 시스템을 구현한다.

## 4. 핵심 아이디어

본 프로젝트의 핵심은 **AI 기반 습도 예측 제어**이다.

일반적인 조건문 방식은 다음과 같다.

| 방식 | 제어 기준 |
|---|---|
| 단순 자동 가습기 | 현재 습도가 기준값보다 낮으면 가습기 ON |
| 본 프로젝트 | AI가 예측한 10분 후 습도를 기준으로 가습기 ON/OFF 결정 |

예를 들어 현재 실내 습도가 43%라고 하더라도, 외부 습도가 낮고 최근 실내 습도가 빠르게 감소하고 있다면 10분 후에는 더 건조해질 가능성이 높다. 이 경우 시스템은 미리 가습기를 작동시킬 수 있다.

반대로 현재 실내 습도가 낮아 보이더라도 외부 습도가 높고 최근 습도가 증가하고 있다면, 시스템은 가습기를 켜지 않고 상태를 유지할 수 있다.

## 5. 전체 시스템 구조

### 시스템 흐름

```text
1. DHT11 또는 DHT22 센서로 실내 온도와 습도를 측정한다.
2. Water Detection Sensor와 PCF8591을 이용하여 물 수위 상태를 측정한다.
3. OpenWeatherMap API를 통해 외부 온도, 외부 습도, 날씨 상태를 가져온다.
4. 최근 습도 변화량을 계산한다.
5. AI 모델이 10분 후 실내 습도를 예측한다.
6. 예측된 습도와 물 수위 상태를 기준으로 가습기 동작 여부를 결정한다.
7. 릴레이 모듈을 통해 초음파 가습 모듈을 ON/OFF 제어한다.
8. 1602 LCD, LED, 부저, Flask 웹 대시보드에 현재 상태를 표시한다.
```

### 시스템 구성도

| 입력 장치 | 처리 장치 | 출력 장치 |
|---|---|---|
| DHT11/DHT22 온습도 센서 | Raspberry Pi | 초음파 가습 모듈 |
| Water Detection Sensor | AI 습도 예측 모델 | 1602 LCD |
| OpenWeatherMap API | Flask Server | LED |
| PCF8591 ADC 모듈 | 제어 알고리즘 | 부저 |
| 최근 습도 변화 데이터 | GPIO 제어 | 웹 대시보드 |

## 6. 주요 기능

### 6.1 실내 온습도 측정

DHT11 또는 DHT22 센서를 사용하여 실내 온도와 습도를 측정한다. 측정된 습도 값은 AI 모델의 입력 데이터로 사용되며, LCD와 웹 대시보드에도 표시된다.

### 6.2 외부 날씨 API 연동

OpenWeatherMap API를 사용하여 외부 온도, 외부 습도, 날씨 상태를 수집한다. 외부 습도와 날씨 정보는 실내 습도 변화 예측에 활용된다.

### 6.3 물 수위 감지

Water Detection Sensor와 PCF8591 AD/DA Converter를 사용하여 물통의 수위 상태를 감지한다. 물 부족 상태가 감지되면 가습기를 강제로 정지시키고, 부저와 LED를 통해 경고한다.

### 6.4 AI 기반 습도 예측

AI 모델은 현재 실내 온습도, 외부 온습도, 최근 습도 변화량, 가습기 상태, 물 수위 상태를 입력받아 10분 후 실내 습도를 예측한다. 이 예측값을 기반으로 가습기 동작 여부를 결정한다.

### 6.5 가습기 자동 제어

릴레이 모듈을 이용하여 초음파 가습 모듈의 전원을 제어한다. 예측 습도가 낮고 물이 충분하면 가습기를 켜고, 습도가 충분하거나 물이 부족하면 가습기를 끈다.

### 6.6 1602 LCD 상태 표시

1602 LCD에 현재 습도, AI 예측 습도, 가습기 상태, 물 부족 상태를 표시한다. 사용자는 별도의 웹 접속 없이도 장치 상태를 바로 확인할 수 있다.

### 6.7 LED 및 부저 알림

LED와 부저를 이용하여 시스템 상태를 직관적으로 표시한다.

| 상태 | LED | 부저 | 가습기 |
|---|---|---|---|
| 정상 | 초록 LED | OFF | OFF |
| 건조 / 가습 필요 | 노랑 LED | OFF | ON |
| 물 부족 | 빨강 LED | ON | OFF |
| 과습 | 초록 또는 빨강 LED | OFF | OFF |

### 6.8 Flask 웹 대시보드

Flask 웹 서버를 통해 현재 실내 온습도, 외부 날씨, 수위 상태, AI 예측 습도, 가습기 동작 상태를 웹페이지에서 확인할 수 있도록 한다.

## 7. AI 모델 설계

### 7.1 AI의 역할

본 프로젝트에서 AI는 단순히 가습기를 켜고 끄는 조건문을 대체하는 것이 아니라, **10분 후 실내 습도를 예측하는 역할**을 수행한다.

시스템은 AI가 예측한 미래 습도 값을 기준으로 가습기 제어 여부를 결정한다.

### 7.2 입력 데이터

AI 모델의 입력 데이터는 다음과 같다.

| 입력 변수 | 설명 |
|---|---|
| indoor_temp | 현재 실내 온도 |
| indoor_humidity | 현재 실내 습도 |
| outdoor_temp | 외부 온도 |
| outdoor_humidity | 외부 습도 |
| humidity_change_5min | 최근 5분간 습도 변화량 |
| humidifier_state | 현재 가습기 상태 |
| water_level | 현재 물 수위 |

### 7.3 출력 데이터

| 출력값 | 설명 |
|---|---|
| predicted_humidity_10min | 10분 후 예측 실내 습도 |

### 7.4 사용 예정 모델

초기 모델은 구현 안정성과 설명 가능성을 고려하여 다음 중 하나를 사용한다.

| 모델 | 특징 |
|---|---|
| Linear Regression | 구현이 쉽고 예측 흐름을 설명하기 좋음 |
| Decision Tree Regressor | 판단 구조를 이해하기 쉬움 |
| Random Forest Regressor | 여러 입력값 간의 비선형 관계를 반영하기 좋음 |

최종적으로는 **Random Forest Regressor**를 우선적으로 적용할 계획이다.

### 7.5 AI 제어 방식

AI가 예측한 습도를 기준으로 다음과 같이 제어한다.

| 예측 결과 | 제어 동작 |
|---|---|
| 물 부족 감지 | 가습기 OFF, 빨강 LED ON, 부저 ON |
| 예측 습도 45% 미만 | 가습기 ON, 노랑 LED ON |
| 예측 습도 45~60% | 가습기 OFF 또는 현재 상태 유지 |
| 예측 습도 60% 초과 | 가습기 OFF, 과습 방지 |

## 8. API 활용 계획

### 8.1 사용 API

**OpenWeatherMap API**

### 8.2 수집 데이터

| 데이터 | 활용 목적 |
|---|---|
| 외부 온도 | 실내 습도 변화 예측 보조 |
| 외부 습도 | 실내 건조 가능성 판단 |
| 날씨 상태 | 비, 맑음, 흐림 등 외부 환경 반영 |
| 지역 정보 | 기본값: Seoul |

### 8.3 API 활용 이유

실내 습도는 외부 환경의 영향을 받을 수 있다. 특히 외부 습도가 낮은 날에는 실내가 더 빠르게 건조해질 수 있고, 비가 오거나 외부 습도가 높은 날에는 실내 습도 변화가 완만할 수 있다.

따라서 외부 날씨 API 데이터를 함께 활용하여 AI 모델이 더 많은 환경 정보를 기반으로 습도 변화를 예측할 수 있도록 한다.

### 시나리오 1: 정상 상태

- 실내 습도와 예측 습도가 적정 범위에 있음
- 가습기 OFF
- 초록 LED ON
- LCD에 NORMAL 표시
- 웹 대시보드에 정상 상태 표시

### 시나리오 2: 건조 상태

- 현재 습도 또는 AI 예측 습도가 낮게 나타남
- 물 수위가 충분함
- 가습기 ON
- 노랑 LED ON
- LCD에 Humidifier ON 표시
- 웹 대시보드에 DRY 또는 HUMIDIFYING 표시

### 시나리오 3: 물 부족 상태

- Water Sensor에서 물 부족 감지
- 가습기 강제 OFF
