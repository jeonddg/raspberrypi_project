# AI 기반 실내 습도 예측 스마트 자동 가습 시스템 기획서

## 1. 프로젝트명

**라즈베리파이를 활용한 AI 기반 실내 습도 예측 스마트 자동 가습 시스템 구현**

영문 제목:

**Development of an AI-based Indoor Humidity Prediction Smart Humidifier System Using Raspberry Pi**

## 2. 프로젝트 개요

본 프로젝트는 라즈베리파이를 이용하여 실내 습도를 자동으로 관리하는 스마트 가습 시스템을 구현하는 것을 목표로 한다.

일반적인 자동 가습기는 현재 습도만 기준으로 가습기를 켜고 끄는 방식이다. 하지만 본 프로젝트에서는 실내 온습도 센서값, 외부 날씨 API 데이터, 최근 습도 변화량, 물 수위 상태를 함께 활용하여 AI 모델이 10분 후 실내 습도를 예측하도록 구성한다.

시스템은 AI가 예측한 미래 습도를 기준으로 가습기 모듈을 자동 제어하며, 물 부족 상태가 감지되면 가습기를 강제로 정지하고 LCD, LED, 부저를 통해 사용자에게 알림을 제공한다.

## 3. 프로젝트 핵심 컨셉

기존 자동 가습기:

```text
현재 습도 낮음 -> 가습기 ON
현재 습도 높음 -> 가습기 OFF
```

본 프로젝트:

```text
실내 온습도 측정
+ 외부 날씨 API 확인
+ 최근 습도 변화량 계산
+ 비접촉식 수위센서로 물 부족 감지
↓
AI 모델이 10분 후 실내 습도 예측
↓
예측 습도를 기준으로 가습기 ON/OFF 제어
↓
LCD, LED, 부저, 웹 대시보드에 상태 표시
```

즉, 단순 조건문 기반 제어가 아니라 **AI 예측 기반 제어**로 차별화한다.

## 4. 주제 선정 이유

이 주제는 실생활에서 바로 활용 가능한 스마트홈/IoT 시스템이다. 단순히 센서값을 읽고 LED를 켜는 실습 수준이 아니라, 실제 가습기 모듈을 제어하여 눈에 보이는 결과물을 만들 수 있다.

또한 과제 조건인 브레드보드와 센서 활용, 지난 수업 내용 2개 이상 활용, 결과보고서·유튜브·깃허브 제출 조건을 모두 자연스럽게 만족한다.

특히 본 프로젝트는 다음 요소를 하나의 목적 안에 통합한다.

| 요소 | 역할 |
|---|---|
| DHT11/DHT22 | 실내 온습도 측정 |
| OpenWeatherMap API | 외부 온습도 데이터 수집 |
| AI 모델 | 10분 후 실내 습도 예측 |
| 비접촉식 수위센서 | 물 부족 감지 |

따라서 과제용으로도 좋고, 포트폴리오에 기록하기에도 적합하다.

### 6-2. 추가 구매 목록

필수 구매:

- 비접촉식 수위센서 XKC-Y25 계열

선택 구매:

- DHT22 온습도 센서
- 로직 레벨 컨버터 또는 저항 10kΩ/20kΩ

DHT11이 이미 있으므로 DHT22는 필수는 아니다. 다만 DHT22를 사용하면 습도 측정 정확도가 올라가고, 보고서에서 “정밀한 습도 측정”이라고 설명하기 좋다.

비접촉식 수위센서는 라즈베리파이 GPIO 입력 전압 문제를 확인해야 한다. 센서 OUT이 5V로 출력되는 제품이면 라즈베리파이에 바로 연결하지 않고 전압 분배 회로나 로직 레벨 컨버터를 사용한다.

## 7. 수위 감지 방식

기존 키트에 있는 Water Detection Sensor는 물에 직접 닿아야 하는 접촉식 센서이다. 이 방식은 짧은 실험에는 사용할 수 있지만, 가습기 물병 안에 장시간 넣어두면 센서 부식, 물 오염, 오작동 가능성이 있다.

따라서 최종 프로젝트에서는 **비접촉식 수위센서**를 사용한다.

비접촉식 수위센서 사용 방식:

```text
물병 외부 하단에 센서 부착
↓
센서 위치까지 물이 있으면 WATER OK
↓
물이 센서 위치 아래로 내려가면 LOW WATER
↓
LOW WATER 상태에서는 가습기 강제 OFF
```

센서는 물병 맨 아래가 아니라, 가습기 필터심이 안정적으로 물을 빨아올릴 수 있는 최소 수위보다 약간 위쪽에 부착한다.

```text
물 충분함 -> 센서 위치에 물 있음 -> 가습 가능
물 부족함 -> 센서 위치보다 물 낮음 -> 가습기 OFF
```

이렇게 하면 가습기 모듈이 물 부족 상태에서 공회전하는 것을 방지할 수 있다.

## 8. AI 기능 설계

### 8-1. AI가 하는 역할

이 프로젝트에서 AI는 단순히 “켜라/꺼라”를 조건문으로 판단하는 것이 아니다.

AI의 핵심 역할은 다음과 같다.

> 현재 실내 환경과 외부 날씨 데이터를 기반으로 10분 후 실내 습도를 예측한다.

즉, 현재 습도만 보는 것이 아니라 앞으로 습도가 떨어질지, 유지될지, 올라갈지를 예측한다.

### 8-2. AI 모델 입력값

| 입력값 | 설명 |
|---|---|
| indoor_temp | 현재 실내 온도 |
| indoor_humidity | 현재 실내 습도 |
| outdoor_temp | 외부 온도 |
| outdoor_humidity | 외부 습도 |
| humidity_change_5min | 최근 5분간 실내 습도 변화량 |
| humidifier_state | 현재 가습기 상태, ON=1/OFF=0 |
| water_state | 물 상태, 물 있음=1/물 부족=0 |

### 8-3. AI 모델 출력값

| 출력값 | 설명 |
|---|---|
| predicted_humidity_10min | 10분 후 예측 실내 습도 |

### 8-4. 사용할 모델

추천 모델:

- RandomForestRegressor

대체 가능 모델:

- DecisionTreeRegressor
- LinearRegression

RandomForestRegressor를 추천하는 이유는 실내 습도 변화처럼 여러 변수의 영향을 받는 데이터를 비교적 안정적으로 예측할 수 있고, 딥러닝보다 구현과 설명이 쉽기 때문이다.

### 8-5. 제어 기준

AI가 예측한 습도를 기준으로 가습기 상태를 결정한다.

| 조건 | 제어 동작 |
|---|---|
| 물 부족 상태 | 가습기 강제 OFF, 빨강 LED ON, 부저 ON |
| 예측 습도 < 45% | 가습기 ON, 노랑 LED ON |
| 예측 습도 45~60% | 가습기 OFF 또는 현재 상태 유지, 초록 LED ON |
| 예측 습도 > 60% | 가습기 OFF, 과습 방지 |

핵심은 현재 습도 기준이 아니라 **AI가 예측한 미래 습도 기준**이라는 점이다.

## 9. API 활용 설계

사용할 API:

- OpenWeatherMap API

수집 데이터:

| 데이터 | 사용 목적 |
|---|---|
| 외부 온도 | 실내 습도 예측 입력값 |
| 외부 습도 | 실내 건조 가능성 판단 |
| 날씨 상태 | Clear, Rain, Clouds 등 외부 환경 참고 |

외부 습도와 날씨는 실내 습도 변화에 영향을 줄 수 있다. 외부가 매우 건조하면 실내 습도도 빠르게 낮아질 가능성이 있고, 비가 오거나 외부 습도가 높으면 실내 습도 감소 속도가 완만할 수 있다.

따라서 외부 날씨 API 데이터를 AI 예측 모델의 입력값으로 사용하여, 단순 실내 센서 기반 제어보다 더 스마트한 제어를 구현한다.

## 10. 1602 LCD 출력 계획

LCD는 프로젝트 완성도를 보여주는 핵심 출력 장치로 사용한다.

기본 화면:

```text
H:42%  P:38%
Humidifier: ON
```

여기서:

- H = 현재 실내 습도
- P = AI 예측 습도

정상 상태:

```text
H:52%  P:53%
Status: NORMAL
```

가습 중:

```text
H:41%  P:38%
Status: HUMID
```

물 부족 상태:

```text
LOW WATER!
Humidifier OFF
```

외부 습도 표시 화면:

```text
In:42% Out:28%
AI Predict:38%
```

LCD에는 화면을 2~3초마다 번갈아 출력하도록 구성하면 좋다.

## 12. 릴레이 제어 방식

초음파 미니 가습기 모듈은 라즈베리파이 GPIO에 직접 연결하지 않는다. 라즈베리파이는 릴레이만 제어하고, 가습기 전원은 외부 5V 전원으로 공급한다.

```text
외부 5V 전원
↓
릴레이 접점
↓
초음파 미니 가습기 모듈
```

제어 구조:

```text
Raspberry Pi GPIO -> 릴레이 IN
릴레이 ON -> 가습기 전원 공급 -> 미스트 발생
릴레이 OFF -> 가습기 전원 차단 -> 미스트 정지
```

이 구조는 라즈베리파이 GPIO 보호와 안정적인 가습기 동작을 위해 필요하다.

| 항목 | 설명 |
|---|---|
| 실내 온도 | DHT 센서 측정값 |
| 실내 습도 | DHT 센서 측정값 |
| 외부 온도 | OpenWeatherMap API |
| 외부 습도 | OpenWeatherMap API |
| 물 상태 | 비접촉식 수위센서 |
| AI 예측 습도 | 10분 후 예측값 |
| 가습기 상태 | ON/OFF |
| 시스템 상태 | NORMAL/HUMIDIFYING/LOW WATER |

웹 대시보드 예시:

```text

Outdoor Humidity   : 28%
Predicted Humidity : 38%
Water State        : OK
Humidifier State   : ON

```

## 14. 전체 동작 흐름

1. DHT 센서가 실내 온도와 습도를 측정한다.
2. 비접촉식 수위센서가 물병 내부의 물 부족 여부를 감지한다.
3. OpenWeatherMap API를 통해 외부 온도와 외부 습도를 가져온다.
4. 최근 5분간 실내 습도 변화량을 계산한다.
5. AI 모델이 10분 후 실내 습도를 예측한다.
6. 물 부족 상태이면 가습기를 무조건 정지한다.
7. 물이 충분하면 AI 예측 습도를 기준으로 가습기를 ON/OFF 한다.
8. LCD에 현재 습도, 예측 습도, 가습기 상태를 표시한다.
9. LED와 부저로 상태를 표시한다.
10. Flask 웹 대시보드에 전체 상태를 표시한다.
