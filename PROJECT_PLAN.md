## 2. 프로젝트 한 줄 요약

실내 온습도와 외부 날씨 데이터를 기반으로 AI가 10분 후 실내 습도를 예측하고, 예측 결과와 비접촉식 수위 감지 결과를 이용해 물병형 초음파 가습기를 자동 제어하는 AIoT 스마트 가습 시스템이다.

## 3. 프로젝트 목적

일반적인 자동 가습기는 현재 습도만 기준으로 가습기를 켜고 끈다. 예를 들어 실내 습도가 45% 미만이면 가습기를 켜고, 60% 이상이면 끄는 방식이다.

하지만 본 프로젝트는 단순 조건문 방식에서 한 단계 확장하여 다음 데이터를 함께 활용한다.

- 실내 온도
- 실내 습도
- 외부 온도
- 외부 습도
- 최근 습도 변화량
- 가습기 작동 상태
- 물 수위 상태

이 데이터를 기반으로 AI 모델이 10분 후 실내 습도를 예측하고, 그 예측값에 따라 가습기를 자동으로 켜거나 끄도록 설계한다.

핵심은 다음과 같다.

```text
현재 습도 기준 제어 X
미래 습도 예측 기반 제어 O
```

## 4. 핵심 컨셉


본 프로젝트 방식:

```text

```

## 7. 비접촉식 수위센서 사용 이유

기존 키트에 있는 판형 Water Detection Sensor는 물에 직접 닿는 접촉식 센서이다. 이 방식은 짧은 실험에는 가능하지만, 가습기 물병 안에 계속 넣어두면 부식, 물 오염, 오작동 가능성이 있다.

그래서 최종 프로젝트에서는 비접촉식 수위센서를 사용한다.

사용 방식:

```text
물병 바깥쪽 하단에 비접촉식 수위센서 부착
        ↓
센서 위치까지 물이 있으면 WATER OK
        ↓
물이 센서 위치 아래로 내려가면 LOW WATER
        ↓
LOW WATER 상태에서는 가습기 강제 OFF
```

센서는 물병 맨 아래가 아니라, 가습기 필터심이 물을 안정적으로 빨아올릴 수 있는 최소 수위보다 약간 위쪽에 붙인다.

```text
물 충분함 -> 센서 위치에 물 있음 -> 가습 가능
물 부족함 -> 센서 위치보다 물 낮음 -> 가습기 OFF
```

이렇게 하면 초음파 가습 모듈이 물 부족 상태에서 공회전하는 것을 막을 수 있다.

## 8. AI 기능 설계

### 8-1. AI가 하는 역할

이 프로젝트에서 AI의 역할은 **10분 후 실내 습도를 예측하는 것**이다.

단순 조건문:

```text
현재 습도 < 45% -> 가습기 ON
현재 습도 >= 45% -> 가습기 OFF
```

본 프로젝트:

```text
현재 실내 온습도
+ 외부 온습도
+ 최근 습도 변화량
+ 가습기 현재 상태
+ 물 상태
        ↓
AI 모델 입력
        ↓
10분 후 실내 습도 예측
        ↓
예측값 기준으로 가습기 제어
```

현재 습도가 46%로 당장은 괜찮아 보여도 외부 습도가 낮고 최근 습도가 계속 떨어지고 있으면 AI가 10분 후 40% 이하로 떨어질 수 있다고 예측할 수 있다. 그러면 시스템은 미리 가습기를 켠다.

반대로 현재 습도가 44%로 낮아도 외부 습도가 높고 최근 습도가 상승 중이면 AI가 10분 후 적정 습도에 도달할 수 있다고 예측할 수 있다. 그러면 가습기를 켜지 않는다.

### 8-2. AI 입력값

| 입력값 | 설명 |
|---|---|
| indoor_temp | 현재 실내 온도 |
| indoor_humidity | 현재 실내 습도 |
| outdoor_temp | 외부 온도 |
| outdoor_humidity | 외부 습도 |
| humidity_change_5min | 최근 5분간 실내 습도 변화량 |
| humidifier_state | 현재 가습기 상태, ON=1/OFF=0 |
| water_state | 물 상태, 물 있음=1/물 부족=0 |

### 8-3. AI 출력값

| 출력값 | 설명 |
|---|---|
| predicted_humidity_10min | 10분 후 예측 실내 습도 |

### 8-4. 사용할 AI 모델

최종 추천:

- RandomForestRegressor

대체 가능:

- DecisionTreeRegressor

## 10. 가습기 제어 로직

AI가 예측한 습도를 기준으로 가습기 상태를 결정한다.

```text
물 부족 상태
-> 가습기 강제 OFF
-> 빨강 LED ON
-> 부저 ON
-> LCD에 LOW WATER 표시

예측 습도 < 45%
-> 가습기 ON
-> 노랑 LED ON
-> LCD에 HUMIDIFIER ON 표시

예측 습도 45~60%
-> 가습기 OFF 또는 현재 상태 유지
-> 초록 LED ON
-> LCD에 NORMAL 표시

예측 습도 > 60%
-> 가습기 OFF
-> 과습 방지
```

핵심은 현재 습도가 아니라 AI 예측 습도 기준으로 제어한다는 점이다.

## 11. 1602 LCD 출력 계획

LCD는 시연 영상에서 프로젝트 완성도를 보여주는 중요한 장치다.

기본 화면:

```text
H:42%  P:38%
Humidifier: ON
```

여기서:

- H = 현재 실내 습도
- P = AI 예측 습도

정상 상태:

```text
H:52%  P:53%
Status: NORMAL
```

가습 중:

```text
H:41%  P:38%
Status: HUMID
```

물 부족 상태:

```text
LOW WATER!
Humidifier OFF
```

외부 습도 표시:

```text
In:42% Out:28%
AI Predict:38%
```

LCD 화면은 2~3초마다 번갈아 출력하면 영상에서 더 좋아 보인다.

## 13. 릴레이 제어 방식

초음파 미니 가습기 모듈은 라즈베리파이 GPIO에 직접 연결하지 않는다. 가습기 모듈은 외부 5V USB 전원을 사용하고, 라즈베리파이는 릴레이만 제어한다.

```text
외부 5V USB 전원
        ↓
릴레이 접점
        ↓
초음파 미니 가습기 모듈
```

제어 구조:

```text
Raspberry Pi GPIO -> 릴레이 IN
릴레이 ON -> 가습기 전원 공급 -> 미스트 발생
릴레이 OFF -> 가습기 전원 차단 -> 미스트 정지
```

이 방식은 라즈베리파이 GPIO 보호와 안정적인 전원 공급을 위해 필요하다.

## 15. 전체 동작 흐름

1. DHT 센서가 실내 온도와 습도를 측정한다.
2. 비접촉식 수위센서가 물병 내부의 물 부족 여부를 감지한다.
3. OpenWeatherMap API를 통해 외부 온도와 외부 습도를 가져온다.
4. 최근 5분간 실내 습도 변화량을 계산한다.
5. AI 모델이 10분 후 실내 습도를 예측한다.
6. 물 부족 상태이면 가습기를 무조건 정지한다.
7. 물이 충분하면 AI 예측 습도를 기준으로 가습기를 ON/OFF 한다.
8. LCD에 현재 습도, 예측 습도, 가습기 상태를 표시한다.
9. LED와 부저로 상태를 표시한다.
10. Flask 웹 대시보드에 전체 상태를 표시한다.

## 19. 시연 영상 촬영 계획

### 19-1. 시연 영상 핵심 전략

실제 시스템은 “10분 후 습도 예측” 구조로 설계하되, 시연 영상에서는 데모 모드를 사용해서 전체 기능을 짧은 시간 안에 보여준다.

영상에서는 다음처럼 설명하면 된다.

> 실제 시스템은 실내 온습도, 외부 날씨 API, 최근 습도 변화량을 기반으로 10분 후 습도를 예측하도록 설계하였다. 시연 영상에서는 모든 기능을 짧은 시간 안에 확인하기 위해 갱신 주기와 기준값을 데모 모드로 조정하여 테스트하였다.

### 19-2. 데모 모드 설계

```python
DEMO_MODE = True

if DEMO_MODE:
    update_interval = 3       # 3초마다 값 갱신
    dry_threshold = 55        # 시연용 기준 습도
    prediction_label = "10min"
else:
    update_interval = 60      # 실제 동작 시 1분마다 갱신
    dry_threshold = 45
    prediction_label = "10min"
```

영상에서는 현재 습도가 실제로 빠르게 변하지 않아도, 기준값이나 테스트 입력값을 조정해 가습기 ON/OFF 장면을 빠르게 보여줄 수 있다.
